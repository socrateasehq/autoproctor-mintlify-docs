Candidates must log in to AutoProctor before they can attempt a test. Authentication is required to uniquely identify each candidate, prevent impersonation, and accurately track test attempts across devices.

## Why Login Is Required

Without login, AutoProctor cannot distinguish between different candidates accessing the same test link. Login enables the system to:

- Associate proctoring data with a specific candidate
- Enforce [maximum attempt limits](/results/maximum-attempts)
- Track [test resumption](/results/resuming-test-attempts) across sessions
- Apply [email-based access restrictions](/results/restricting-by-email)

## Available Login Methods

AutoProctor offers three authentication options:

1. **Sign in with Google** -- Uses the candidate's Google account
2. **Sign in with Microsoft** -- Uses the candidate's Microsoft account
3. **Sign in with Email** -- Sends a one-time verification code to the candidate's email

## Login Methods by Test Type

The available login methods depend on the type of test you are administering.

| Test Type | Available Login Methods |
|---|---|
| Socratease Quizzes | Google, Microsoft, Email |
| Google Forms | Google only |
| Microsoft Forms | Microsoft only |
| IFrame Tests | Google, Microsoft, Email |

<Warning>
Google Forms tests require candidates to sign in with Google, and Microsoft Forms tests require Microsoft sign-in. If your candidates only have one type of account, choose your quiz platform accordingly.
</Warning>

## Impact on Attempt Tracking

AutoProctor tracks test attempts by the email address used to sign in. If a candidate signs in with different email addresses (for example, using Google sign-in one time and Email sign-in another time), each email is treated as a separate candidate. This can bypass [maximum attempt limits](/results/maximum-attempts).

To prevent this, consider restricting login to a single method by choosing a test type that supports only one sign-in option, or by using [email-based restrictions](/results/restricting-by-email).
