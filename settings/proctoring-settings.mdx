Proctoring Settings control what AutoProctor monitors during a test — from tab switching and camera detection to full screen enforcement and random photo capture. These settings determine how strictly candidates are supervised.

<Note>
To enable proctoring, you must check the **Enable Proctor** checkbox in the Main Settings section. If proctoring is disabled, none of these settings apply.
</Note>

## Basic Proctoring Options

| Setting | What It Does |
|---|---|
| **Tab Switching** | Detects when a candidate switches to a different browser tab or application. Captures a screenshot of the tab they switched to. |
| **Detect Multiple Monitors** | Detects if a candidate has connected external monitors to their computer. |
| **Camera** | Detects if no face is visible on the camera feed, or if multiple faces are visible. Captures photos as evidence. |
| **Microphone** | Monitors the sound environment and records audio when noise is detected. |
| **Photos at Random** | Captures a few random photos throughout the test at randomized intervals. |
| **Enforce Full Screen** | Forces candidates to take the test in full screen mode. Exiting full screen is recorded as a violation. |
| **Enforce Desktop** | Requires candidates to use a desktop or laptop computer. The test will not load on tablets or mobile devices. |
| **Photo Before Test Start** | Captures a photo of the candidate before the test begins. |
| **Customize Message** | Customize the message candidates see when asked to take a photo before the test. Use this to ask candidates to display their ID card. |

<Note>
The **Customize Message** option is a Premium feature.
</Note>

## Enhanced Proctoring

Enhanced proctoring adds advanced anti-cheating measures. Each enhanced feature requires **4 additional credits per attempt**.

<Frame caption="Enhanced proctoring settings panel.">
  <img src="/images/creating-tests/enhanced-proctoring.png" alt="Enhanced proctoring settings panel" />
</Frame>

| Feature | What It Does | Credits |
|---|---|---|
| **ID Card Verification** | Candidate uploads an ID card (student ID, passport, driver's license, government ID) with their photo and name visible. AutoProctor verifies this against the Photo Before Test Start and the name they entered. | 4 per attempt |
| **Impersonation Detection** | Detects if someone other than the original candidate is taking the test. | 4 per attempt |
| **Auxiliary Device (360°)** | Candidate pairs their phone to their laptop for 360-degree proctoring. Also helps detect AI tools that display answers as screen overlays. | 4 per attempt |
| **Session Recording** | Records the candidate's screen and actions, including mouse clicks and keyboard input. | 4 per attempt |

## Communication Settings

These settings control how violation evidence is handled:

- **Me after the Test** — Store evidence (photos, audio, screenshots) for later review. Keep this enabled unless you have a specific reason to disable it.
- **User during the Test** — Notify the candidate during the test when a violation is detected. This helps candidates fix harmless issues (like background music triggering a noise violation).
